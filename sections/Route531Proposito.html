1. Propósito del Documento

El presente documento tiene como objetivo establecer el modelo de operación, gobierno y responsabilidades del servicio de DNS basado en Amazon Route 53, definiendo de forma clara la interacción entre los equipos de Public Cloud (AWS), Network Services, Network Platform – NetFactory y Seguridad de la Información (CISO).

Este documento busca asegurar que el servicio de DNS sea operado de manera segura, estable, escalable y auditable, mediante la correcta separación de funciones entre operación, ingeniería, automatización y gobierno, alineándose con las mejores prácticas de ITIL y los lineamientos internos de la organización.

Asimismo, establece los principios y lineamientos necesarios para:

  • Garantizar la continuidad operativa del servicio DNS.
  • Reducir riesgos técnicos y operativos derivados de cambios o configuraciones incorrectas.
  • Facilitar la transición desde un modelo centralizado en el equipo de Public Cloud (AWS) hacia un modelo distribuido y gobernado.
  • Asegurar la trazabilidad, control y cumplimiento de los procesos relacionados con el servicio.

Este documento no define configuraciones técnicas específicas, sino que establece el marco de gobierno, operación y responsabilidades bajo el cual se gestionará el servicio DNS dentro de la organización.
