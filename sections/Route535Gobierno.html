5. Modelo de Gobierno

5.1 Objetivo del modelo de gobierno

El modelo de gobierno del servicio DNS tiene como objetivo garantizar que la toma de decisiones, la operación y la evolución del servicio se realicen de manera controlada, segura y alineada con los objetivos del negocio, asegurando:

  • Separación clara de responsabilidades.
  • Control efectivo del riesgo operativo y de seguridad.
  • Trazabilidad de decisiones y cambios.
  • Cumplimiento de políticas internas y buenas prácticas (ITIL).

5.2 Principios de gobierno

  • Separación de funciones: ninguna función crítica debe concentrarse en un solo equipo.
  • Responsabilidad clara: cada actividad tiene un responsable definido.
  • Gobierno sobre operación: las decisiones estratégicas no se ejecutan de forma operativa.
  • Trazabilidad: todas las acciones deben ser auditables.
  • Automatización controlada: priorización de automatización bajo estándares definidos.
  • Gestión de riesgo integrada: los riesgos se evalúan antes de ejecutar cambios relevantes.

5.3 Estructura de gobierno

El gobierno del servicio se ejerce mediante la interacción coordinada de los siguientes actores:

Public Cloud (AWS)

  • Gobierna la plataforma tecnológica.
  • Define y aplica controles sobre la infraestructura DNS.
  • Garantiza la disponibilidad del servicio base.

Network Platform – Network Services

  • Define la arquitectura del servicio DNS.
  • Establece estándares técnicos y patrones de diseño.
  • Aprueba cambios técnicos de impacto medio y alto.
  • Evalúa implicaciones técnicas y de escalabilidad.

IT Ops – Network Services

  • Ejecuta la operación diaria conforme a los lineamientos establecidos.
  • Atiende incidentes operativos.
  • Escala situaciones que exceden su ámbito de control.

Network Platform – NetFactory

  • Implementa automatizaciones y despliegues.
  • Garantiza consistencia técnica y repetibilidad.
  • Apoya los procesos de transición y mejora continua.

CISO

  • Define políticas de seguridad aplicables al servicio.
  • Aprueba cambios de alto impacto.
  • Supervisa el cumplimiento normativo.
  • Gestiona excepciones de seguridad.

5.4 Modelo de toma de decisiones

Tipo de decisión Responsable Participación
Estratégica CISO Public Cloud / Network Platform
Técnica (arquitectura) Network Platform Public Cloud / NetFactory
Operativa IT Ops Network Platform
Automatización NetFactory Network Platform
Seguridad CISO Todos

5.5 Mecanismos de control

  • Flujos formales de aprobación mediante ITSM.
  • Registro y trazabilidad de cambios.
  • Evidencia documentada para auditoría.
  • Revisión periódica de controles y procesos.
  • Evaluación de riesgos ante cambios relevantes.

5.6 Escalamiento y resolución de conflictos

Cuando exista un conflicto operativo, técnico o de gobierno, se seguirá el siguiente esquema de escalamiento:

  1. IT Ops – Network Services intentará resolver el incidente dentro de su ámbito operativo.
  2. En caso de requerir decisiones técnicas o de arquitectura, se escalará a Network Platform – Network Services.
  3. Si el evento impacta la plataforma o excede los límites definidos, se involucrará a Public Cloud (AWS).
  4. En situaciones que representen riesgo de seguridad, cumplimiento o impacto crítico al negocio, se escalará al CISO, quien definirá la acción a seguir.
