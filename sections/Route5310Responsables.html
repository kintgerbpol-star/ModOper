10. Aprobaciones y Responsables

10.1 Objetivo

Definir claramente los niveles de aprobación, responsabilidad y autoridad asociados a la operación del servicio DNS, garantizando que las decisiones se tomen de forma controlada, trazable y alineada con el modelo de gobierno establecido.

10.2 Principios generales

Las aprobaciones relacionadas con el servicio DNS se rigen por los siguientes principios:

  • Separación de funciones: quien ejecuta no debe aprobar su propio trabajo.
  • Trazabilidad: toda aprobación debe quedar registrada.
  • Proporcionalidad: el nivel de aprobación debe corresponder al impacto del cambio o acción.
  • Responsabilidad clara: cada decisión tiene un dueño definido.
  • Gobierno centralizado: las decisiones críticas siguen un modelo formal de aprobación.

10.3 Niveles de aprobación

Nivel 1 – Operativo

  • Aplica a actividades rutinarias y de bajo riesgo.
  • Incluye solicitudes estándar y cambios operativos.

Autoridad de aprobación:

  • Network Platform – Network Services

Nivel 2 – Técnico / Arquitectura

  • Aplica a cambios con impacto técnico relevante.
  • Incluye modificaciones de arquitectura o comportamiento del servicio.

Autoridad de aprobación:

  • Network Platform – Network Services
  • CISO (cuando exista impacto en seguridad)

Nivel 3 – Riesgo / Seguridad

  • Aplica a cambios con impacto alto o riesgo significativo.
  • Incluye excepciones de seguridad y desviaciones de control.

Autoridad de aprobación:

  • CISO

10.4 Matriz de aprobación por tipo de actividad

Actividad Aprobador Primario Aprobador Secundario
Cambios estándar Network Platform – Network Services —
Cambios complejos / arquitectura Network Platform – Network Services CISO
Automatización – diseño y desarrollo Network Platform – Network Services —
Automatización – ejecución operativa Network Platform – Network Services —
Gestión de incidentes Incident Management CISO (si aplica)
Gestión de seguridad CISO —
Excepciones de seguridad CISO —
Auditorías (respuesta y remediación) Network Platform – Network Services CISO

10.5 Mecanismo de aprobación

  • Las aprobaciones deben registrarse mediante ServiceNow o correo electrónico institucional, según aplique.
  • Toda aprobación debe:
    • Identificar claramente al aprobador.
    • Estar asociada a una solicitud o evento específico.
    • Ser verificable para fines de auditoría.
  • En caso de indisponibilidad temporal de ServiceNow, el correo institucional será el medio alternativo válido.
